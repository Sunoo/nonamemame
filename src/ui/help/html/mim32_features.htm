Install Procedures

To Install MadeInMame32 Manually

Download MadeInMame32 archive, then :

·         New install :

       Download and install Mame32 as described here.

       Before starting Mame32, replace Mame32.exe and Mame32.chm with MadeInMAME32.exe and MadeInMAME32.chm.

 

·         Install on an existing Mame32 release :

       Replace Mame32.exe and Mame32.chm with MadeInMAME32.exe and MadeInMAME32.chm.

       Rename Mame32ui.ini to MadeInMAME32ui.ini, and ini/Mame32.ini to ini/MadeInMAME32.ini.

 

Thats all. MadeInMAME32 will be run with default parameters.

Visual elements

 

When Mame32 starts, an icon is added in the system tray area : 

  • When clicking the icon with the mouse right button, a popup menu is displayed, allowing to control some Mame32 usual commands ;
     
  • When double-clicking the icon with the mouse left button, the current select game will be launched, even if the main windows is hidden (its name is given by the system tray icon tool tip).

 

Mame32 allows to prevent kids to play erotic or violent games by locking games. Only unlocked games can be played. Playing locked games or unlocking games is submitted to a password successful verification. See Options Menu below for password definition and rules.

·         Each game can be locked/unlocked separately from list window with File Menu or local Context Menu ;

  • An entire folder and its sub folders can be locked or unlocked with local Context Menu.

 

The password state is indicated in the Status Bar 3rd pane :

·         if enabled and verified, an open lock is displayed ;

·         if enabled but not verified, a closed lock is display ;

  • if not enabled, the pane remains empty.

 

The game state is indicated in the Status Bar 2nd pane :

·         if password is enabled and verified, a open lock is displayed for usually locked games (game can be played) ;

·         if password is enabled but not verified, a closed lock is displayed for usually locked games, and an open lock is displayed for usually unlocked games (game cant be played) ;

  • if password is disabled, a key is displayed for usually locked games, and nothing is displayed for usually unlocked games (game can be played).

 

The locked/unlocked games list is stored encrypted in the disk in games.lst file, in the same directory than Mame32 executable.

·         if the list is either not found or corrupt, a default list with all games locked is created ;

  • if the list is found and correctly loaded, new games (e.g. from a new Mame32 release) are considered as unlocked.

 

File Menu

In addition to Mame32s File Menu, the following items are available :

 

       Lock this game : Selecting this option will lock the selected game. Once locked, the game cant be played until the password is verified or disabled.

       Unlock this game : Selecting this option will unlock the selected game. Once unlocked, the game can be played whatever the password state.

       View PCB Info : Selecting this option will open a window and display the selected game PCB information if available. The PCB info shall be in the pcbinfo/ directory. It is allowed to store all files in a single PCBINFO.ZIP archive..

       Save current game list : Selecting this option will create a formatted (with | separators or tabulations) text file with the game list as displayed in the GUI : same columns, same order and same filters. This option is available with DETAIL or GROUPED view mode.

 

Options Menu

In addition to Mame32s File Option, the following items are available :

 

       Password : Selecting this option from the options menu will allow Mame32 password management. The password allow a Mame32 administrator to forbid users to play one or several games. The following rules apply :

-  the password is case-sensitive and must be at least 3 characters and at most 10 characters ;

-  the password is stored encrypted in MadeInMAME32ui.ini file ;

-  when installing Mame32 in a new directory, a default password is created which is enabled ;

-  if the password is corrupt, the default password is automatically created and enabled ;

-  when starting Mame32, the password isnt verified, but its enabling state is kept from previous Mame32 session.

-  when enabling the password, it is not verified, even if it was previously verified ;

-  when the password is verified, it is necessary to quit MadeInMame32 or enter a wrong password to un-verify it ;

-  since it is necessary to know the password, changing it will verify it ;

-  when password is enabled, playing any game (locked or not) with of MadeInMAME32 command line is forbidden. To verify password with command line, just add Pwd=XXX (with no space) as last argument of command line, where XXX is the password.

See Visual Elements above for locking / unlocking games.

       Background Image : Selecting this option from the options menu allows the choosing of a background image from the \bkground directory.  This image will be displayed beneath the three window panes in the GUI and the PCB info.
